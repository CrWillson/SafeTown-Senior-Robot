Detailed Description

Handles the publishing of and subcribing to events between functionally independent components.

Member Function Documentation

◆ processEvents()

void EventManager::processEvents ( )

Begins the event processing loop. Should be executed on the second core to prevent the main loop from blocking.

This function starts an infinite loop and will never return.

◆ publish()

template<typename EventType>
void EventManager::publish ( const EventType & event)
inline

Publish an event to the event queue. All call-back functions registered to that event type will be executed in the order that they were registered.

If no call-back functions are registered to the event type passed, the event will be discarded.

Parameters
event- The event to publish
